# 长安大学校园服务平台 - 本地部署说明

## 项目概述

长安大学校园服务平台是一个面向长安大学师生的综合性校园服务平台，包含用户端和后台管理端，提供东门小吃摊点单、校园超市点单、家教板块、二手交易、驾校板块、学习资料、校园论坛等7大核心功能。

## 技术栈

### 后端
- Node.js (v16+)
- Express.js
- JWT (身份验证)
- CORS (跨域支持)

### 前端
- Vue 3
- Element Plus
- Vite
- Pinia (状态管理)
- Vue Router
- Axios (HTTP请求)

## 环境要求

### 必需软件
1. **Node.js**: v16.0.0 或更高版本

## 部署步骤

### 1. 克隆或下载项目

```bash
# 如果使用 Git
git clone <项目地址>
cd 长安大学校园服务平台

# 或者直接下载并解压项目文件
```

### 2. 后端部署

#### 2.1 安装依赖

```bash
cd backend
npm install
```

#### 2.2 配置环境变量

编辑 `.env` 文件，配置以下参数：

```env
PORT=3000
JWT_SECRET=chd_campus_secret_key_2024
JWT_EXPIRES_IN=7d
EMAIL_HOST=smtp.chd.edu.cn
EMAIL_PORT=587
EMAIL_USER=noreply@chd.edu.cn
EMAIL_PASS=
FRONTEND_URL=http://localhost:5173
```

**配置说明：**
- `PORT`: 后端服务端口，默认 3000
- `JWT_SECRET`: JWT 密钥，建议修改为随机字符串
- `JWT_EXPIRES_IN`: Token 有效期，默认 7 天
- `EMAIL_HOST`: 邮件服务器地址（当前为模拟）
- `EMAIL_PORT`: 邮件服务器端口
- `EMAIL_USER`: 邮件发送账号
- `EMAIL_PASS`: 邮件发送密码
- `FRONTEND_URL`: 前端地址

#### 2.3 启动后端服务

```bash
# 开发模式（支持热重载）
npm run dev

# 生产模式
npm start
```

启动成功后，会看到以下输出：
```
服务器运行在端口 3000
```

#### 2.4 验证后端服务

在浏览器中访问：
```
http://localhost:3000/api/health
```

返回：
```json
{
  "status": "ok",
  "message": "长安大学校园服务平台后端运行正常"
}
```

### 3. 前端部署

#### 3.1 安装依赖

```bash
cd frontend
npm install
```

#### 3.2 启动前端服务

```bash
# 开发模式
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview
```

启动成功后，会看到以下输出：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

#### 3.3 访问前端

在浏览器中访问：
```
http://localhost:5173
```

### 4. 完整启动流程

为了方便，可以同时启动前后端服务：

**方式一：使用两个终端窗口**

```bash
# 终端1 - 启动后端
cd backend
npm run dev

# 终端2 - 启动前端
cd frontend
npm run dev
```

**方式二：使用 concurrently（推荐）**

在项目根目录创建 `package.json`：

```json
{
  "name": "chd-campus-platform",
  "scripts": {
    "dev": "concurrently \"cd backend && npm run dev\" \"cd frontend && npm run dev\"",
    "install:all": "cd backend && npm install && cd ../frontend && npm install"
  }
}
```

安装依赖：
```bash
npm install -g concurrently
npm run install:all
```

启动项目：
```bash
npm run dev
```

## 常见问题

### 1. 端口被占用

**问题：** 启动服务时提示端口被占用

**解决方案：**
```bash
# Windows 查找占用端口的进程
netstat -ano | findstr :3000
# 结束进程
taskkill /PID <进程ID> /F

# 或修改 .env 文件中的 PORT 配置
PORT=3001
```

### 2. npm install 失败

**问题：** 安装依赖时出现错误

**解决方案：**
```bash
# 清除 npm 缓存
npm cache clean --force

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

### 3. 前端无法访问后端

**问题：** 前端请求后端接口失败

**解决方案：**
- 确认后端服务已启动
- 检查 `vite.config.js` 中的代理配置
- 确认 `.env` 中的 `FRONTEND_URL` 配置正确

### 4. 登录失败

**问题：** 登录时提示认证失败

**解决方案：**
- 确认使用的是测试账号
- 检查后端控制台是否有错误信息
- 确认 `.env` 中的 `JWT_SECRET` 配置正确

### 5. 图片无法显示

**问题：** 商品/菜品图片无法显示

**解决方案：**
- 确认 `backend/uploads` 目录存在
- 检查图片 URL 路径是否正确
- 确认后端静态文件服务配置正确

### 6. 邮箱验证码收不到

**问题：** 注册时收不到验证码

**说明：** 当前使用模拟数据，验证码会在后端控制台输出

**解决方案：**
- 查看后端控制台输出的验证码
- 实际部署时需要配置真实的邮件服务

## 生产环境部署

### 1. 后端部署

使用 PM2 进行进程管理：

```bash
# 安装 PM2
npm install -g pm2

# 启动后端
cd backend
pm2 start app.js --name chd-campus-backend

# 查看日志
pm2 logs chd-campus-backend

# 重启服务
pm2 restart chd-campus-backend

# 停止服务
pm2 stop chd-campus-backend
```

### 2. 前端部署

构建生产版本：

```bash
cd frontend
npm run build
```

将 `dist` 目录部署到 Nginx 或其他 Web 服务器：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 安全建议

1. **修改默认密钥**
   - 修改 `.env` 中的 `JWT_SECRET` 为强随机字符串

2. **启用 HTTPS**
   - 生产环境使用 HTTPS 协议

3. **配置 CORS**
   - 限制允许的域名

4. **输入验证**
   - 所有用户输入都进行验证和过滤

5. **错误处理**
   - 不要在生产环境暴露详细的错误信息

## 性能优化

1. **前端优化**
   - 启用代码分割
   - 使用 CDN 加速静态资源
   - 启用 Gzip 压缩

2. **后端优化**
   - 使用缓存减少数据库查询
   - 启用请求压缩
   - 使用连接池管理数据库连接
