# 测试指南

本仓库启动时默认没有自动化测试。下面的步骤可以帮助你搭建并运行前后端的基本测试套件。

---

## 后端（Node/Express）

1. 在 `backend` 目录安装依赖：
   ```bash
   cd backend
   npm install
   ```
2. 已新增的开发依赖：
   - `jest`：测试运行器
   - `supertest`：HTTP 请求模拟
   - `cross-env`：设置环境变量

3. 修改了 `app.js`：
   - 导出 `createApp()` 函数，便于测试时构建实例。
   - 启动逻辑与测试逻辑分离。

4. 新增配置文件 `jest.config.js`，测试匹配路径为 `backend/tests/**/*.test.js`。

5. 示例测试文件：
   - `backend/tests/health.test.js`，验证 `/api/health` 路由和 404 响应。

6. 运行测试：
   ```bash
   npm run test
   ```

> 注意：当前测试并不会连接数据库，若需覆盖需要引入 mock 或搭建专用测试库。

---

## 前端（Vue 3 + Vite）

1. 在 `frontend` 目录安装依赖：
   ```bash
   cd frontend
   npm install
   ```

2. 已新增的开发依赖：
   - `vitest`：Vite 原生测试框架
   - `@testing-library/vue`：Vue 组件渲染/断言
   - `@testing-library/jest-dom`：DOM 断言扩展

3. 在 `vite.config.js` 中添加了 `test` 块，用于配置 Vitest（`jsdom` 环境、全局 `jest-dom`）。

4. 创建了 `src/setupTests.js` 来引入 `jest-dom`。

5. 示例测试：`frontend/tests/Home.spec.js`，检查首页 `<h1>` 是否渲染。

6. 运行测试：
   ```bash
   npm run test
   ```

---

## 进一步扩展

- 后端可以对各个路由、控制器、数据库适配器撰写更多单元/集成测试。
- 前端可以增加组件、store、路由等测试；也可以配置端到端测试（如 Cypress）。
- 考虑在 CI/CD 中加入 `npm run test` 步骤，并使用覆盖率工具。
